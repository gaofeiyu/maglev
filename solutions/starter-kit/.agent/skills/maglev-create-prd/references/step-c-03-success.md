## 成功发现序列：

### 1. 开始成功定义对话

**检查输入文档中的成功指标：**
分析产品简报、研究和头脑风暴文档中已提到了哪些成功标准。

**如果输入文档包含成功标准：**
引导用户完善现有的成功标准：
- 确认材料中已记录的内容
- 从简报、研究和头脑风暴中提取关键成功主题
- 帮助用户识别差距和扩展领域
- 探究具体的、可衡量的结果：用户何时感到高兴/如释重负/获得授权？
- 询问情感成功时刻和完成场景
- 探索超出已捕获内容的“值得”意味着什么

**如果输入文档中没有成功标准：**
从以用户为中心的成功探索开始：
- 引导对话定义什么对用户来说意味着“值得”
- 询问用户意识到他们的问题得到解决的那一刻
- 探索特定的用户结果和情感状态
- 识别成功“顿悟”时刻和完成场景
- 首先关注成功的用户体验

### 2. 探索用户成功指标

倾听具体的用户结果并帮助使其可测量：

- 引导从模糊到具体：不是“用户很高兴” → “用户在 [时间范围] 内完成 [关键行动]”
- 询问情感成功：“他们何时感到高兴/如释重负/获得授权？”
- 识别成功时刻：“什么是‘顿悟’时刻？”
- 定义完成场景：“对用户来说‘完成’是什么样子的？”

### 3. 定义业务成功

过渡到业务指标：
- 引导对话转向业务成功视角
- 探索时间表：3 个月的成功是什么样子的？12 个月呢？
- 识别关键业务指标：收入、用户增长、参与度或其他衡量标准？
- 询问什么具体指标表明“这行得通”
- 从他们的角度理解业务成功

### 4. 挑战模糊指标

推动业务指标的具体化：

- "10,000 个用户" → "什么样的用户？在做什么？"
- "99.9% 正常运行时间" → "真正的担忧是什么 - 数据丢失？支付失败？"
- "快" → "有多快，具体什么需要快？"
- "良好的采用率" → "何时达到百分之多少的采用率？"

### 5. 连接到产品差异化因素

将成功指标与产品的独特之处联系起来：
- 将成功标准连接到产品的独特差异化因素
- 确保指标反映具体的价值主张
- 使成功标准适应领域背景：
  - 消费者：用户喜爱、参与度、留存率
  - B2B：ROI、效率、采用率
  - 开发者工具：开发者体验、社区
  -受监管：合规性、安全性、验证
  - 政府科技：政府合规性、无障碍、采购

### 6. 智能范围谈判

通过成功视角引导范围定义：
- 帮助用户区分 MVP（必须工作才能有用）与增长（竞争力）和愿景（梦想）
- 通过三个范围级别引导对话：
  1. MVP：证明概念必不可少的是什么？
  2. 增长：什么使其具有竞争力？
  3. 愿景：梦想版本是什么？
- 通过对话挑战范围蔓延：这可以等到发布之后吗？这对 MVP 是必不可少的吗？
- 对于复杂领域：确保 MVP 中包含合规性最低要求

### 7. 生成成功标准内容

准备要追加到文档的内容：

#### 内容结构：

保存到文档时，追加这些 2 级和 3 级部分：

```markdown
## 成功标准

### 用户成功

[基于对话的用户成功标准内容]

### 业务成功

[基于对话的业务成功指标内容]

### 技术成功

[基于对话的技术成功要求内容]

### 可衡量结果

[以对话为基础的关于具体可衡量结果的内容]

## 产品范围

### MVP - 最小可行产品

[基于对话的 MVP 范围内容]

### 增长功能 (MVP 后)

[基于对话的增长功能内容]

### 愿景 (未来)

[基于对话的未来愿景内容]
```
