# 2024-2025 Agentic Trends (前沿趋势)

本文档收录了 2024-2025 年 AI Software Engineering 领域的最新突破，重点关注 **"Flow Engineering"** 和 **"Autonomous Developers"**。

## 1. The Rise of Flow Engineering

### **[2401.08500] Code Generation with AlphaCodium: From Prompt Engineering to Flow Engineering**
- **来源**: CodiumAI (Jan 2024)
- **核心理念**: **"Flow Engineering > Prompt Engineering"**。
  - 对于复杂代码任务，单一的高质量 Prompt (Zero-shot/Few-shot) 往往无效。
  - 需要一个 **"Test-based, Multi-stage Flow"**：先生成测试，根据测试生成代码，再根据报错迭代修复。
- **Maglev 借鉴点**:
  - 这是 Maglev **`thinking/` -> `solutions/`** 流程的最强理论背书。
  - 我们不应该追求 "One-Shot" 生成完美代码，而应该构建 **"Iterative Repair Loop"** (对应我们的 Efficiency Matrix Quadrant 2)。
- **推荐指数**: ⭐⭐⭐⭐⭐

## 2. Agentic vs Agentless

### **[2407.01489] Agentless: Demystifying LLM-based Software Engineering Agents**
- **来源**: University of Illinois (Jul 2024)
- **核心理念**: **"Simplicity Wins"**。
  - 复杂的 ReAct 循环容易陷入死循环或 Token 爆炸。
  - 提出一种简单的三阶段流：**Localization (定位) -> Repair (修复) -> Validation (验证)**，不给 LLM 过多的自主权。
  - 在 SWE-bench Lite 上以极低成本击败了许多复杂 Agent。
- **Maglev 借鉴点**:
  - 警示我们不要过度设计 Skill。Maglev 的 Skill 应该原子化、确定性强（类似 Agentless 的 Approach），而不是让 AI 无限发散。
- **推荐指数**: ⭐⭐⭐⭐

### **[2403.06905] AutoDev: Automated AI-Driven Development**
- **来源**: Microsoft (Mar 2024)
- **核心理念**: **"IDE Integrated Agents"**。
  - Agent 不仅仅是对话框里的助手，而是 IDE 的“幽灵用户”，可以触发 Build, Run, Test, Git Commit 等一系列动作。
- **Maglev 借鉴点**:
  - 这就是 Maglev **"Efficiency Matrix"** 的终极形态：Agent 直接操作 IDE 里的各类工具，实现 Spec 到 Code 的无感流转。

## 3. The New Standard: SWE-bench

### **SWE-bench Verified (2024)**
- **来源**: OpenAI / Princeton
- **核心理念**: **"Repo-Level Evaluation"**。
  - 传统的 HumanEval 只测单函数。SWE-bench 测的是“在一个真实仓库里解一个真实 Bug”。
- **Maglev 借鉴点**:
  - 我们的 `eff.bug_pilot` (Bug Pilot Skill) 应该以通过 SWE-bench 分数为考核目标。
  - 强调了 **Rank Retrieval (RAG)** 在大型代码库中的极端重要性。
